Workflow
  1. Individual Registration

    During registration, users specify their home location (town, city, village, postal code, etc). Based on their location, the system determines each of their governing entities (country, state, county, town/city etc)

  2. Government Body Registration

    An user may complete the registration for their local governing entities: (town, city, village, county, state, etc). When user register for the same location, they will see the data already entered. They can vote on the accuracy of any data items and specify alternate values. Votes will accumulate for each data value and the values with the most votes become the values that are on the official registration form that is used by the system. Github issue #62

  3. Import recordings or transcripts
    • Download existing online recordings or transcripts on a regular schedule from the online locations specified in the Government Body Registration.
    • User upload of recordings or transcripts. Occasionally the upload process may require manual steps by a volunteer.
    • Many places neither provide transcripts nor recordings of their meetings. Govmeeting will provide a smartphome app that can be used to personally record and upload a meeting recording. Github issue #18
  4. Conversion to text
    • Convert transcripts to plain text Often existing transcripts are in other formats like PDF. These are converted to plain text.
    • Convert recordings using speech-to-text
      The following processing is currently being done:
      • Convert to each of the primary web video formats (mp4, ogg and webm.) This enables volunteers to view or hear the recording during the proofreading stage, irregardless of the device they are using.
      • Extract and merge the audio tracks.
      • Upload the audio file to the Google Cloud.
      • Call the asynchronous Google Speech API to do auto voice recognition.
  5. Further processing of speech-to-text output
    • Speaker change recognition
    • Add speaker identification
    • Proofread generated text
      The following processing is currently being done:
      • Split the video, audio and trancript files into 3 minute work segments.
      • Provide a user friendly interface for quickly correcting errors. See: Proofread
  6. Add section labels
    • Extract information such as speaker and section names. *
    • Put the information into a JSON format for further processing. *
  7. Add issue tags
  8. Populate database with final output
  9. Data is now available for use
    • A browe-able & search-able transcript is now available
    • A summary of the issues discussed at the meeting is sent to those requesting it.
    • Alerts are sent on specific issues to those requesting it.
  10. Virtual meeting is arranged.
    • An agenda is created based on issues at the real meeting.
    • Invitatations are sent to community members. Included are requests for possible additional agenda items.
    • When responses are received, a ballot is sent to those who wil attend. On this ballot, members can vote on whether to include any of the suggested new agenda items.
    • Within a few days, a virtual meeting is held.
